## 작업: GitHub/GitLab 미러 + 언어별 커밋 워크플로 (이슈 #1)

### 이슈
- GitHub #1: Set up GitHub/GitLab mirror remotes and bilingual commit workflow

### 할 내용 (계획)
- GitHub(`cuter74-dev/wowBoard`)·GitLab(`gitlab.oopnwow.com/cuter74/wowboard`) 두 원격 연결.
- **영어 메시지 → GitHub, 한글 메시지 → GitLab**, 항상 둘 다 푸시.
- 매 커밋을 일관되게 하는 헬퍼 스크립트 작성.

### 어떻게 개발/처리했는지
- 토큰은 루트 `.env`의 `GITHUB_TOKEN`/`GITLAB_TOKEN` 사용(.gitignore로 제외). 두 저장소 모두 비어 있어 충돌 없이 첫 푸시.
- **병렬 브랜치 모델**: `gh`(영어 히스토리) → GitHub `main`, `gl`(한글 히스토리) → GitLab `main`. 두 브랜치는 트리(내용) 동일, 메시지/히스토리만 다름.
  - `gl`은 매번 `git read-tree -u --reset gh`로 `gh`의 트리를 그대로 미러링한 뒤 한글 메시지로 커밋.
- `scripts/dual-commit.sh "EN" "KO"` 한 명령으로 EN→GitHub, KO→GitLab 커밋·푸시. (Co-Authored-By 트레일러 자동 추가)
- 이후 모든 작업 커밋은 이 스크립트로 수행.
