## 작업: 계정 간 템플릿 공유 (링크 + 가져오기) (이슈 #4)

### 이슈
- GitHub #4: Share user templates across accounts via link + import

### 할 내용 (계획)
- 템플릿에 공유 토큰 발급 → 다른 계정이 링크로 미리보고 자기 템플릿으로 복사(import).
- Prisma `Template.shareToken`(unique, nullable) + 마이그레이션.
- 백엔드: `POST/DELETE /templates/:id/share`(소유자), `GET /templates/shared/:token`(공개 미리보기), `POST /templates/import/:token`(인증, 복사).
- 프론트: 내 템플릿 행에 🔗 공유(링크 복사), `/template/:token` 미리보기 페이지 + "내 템플릿으로 가져오기".

### 어떻게 개발/처리했는지
- Prisma `Template.shareToken`(unique, nullable). migrate dev가 비TTY에서 막혀 마이그레이션 SQL 수동 생성 후 `prisma migrate deploy`로 적용.
- 백엔드 templates 컨트롤러를 메서드별 가드로 전환(공개 미리보기 허용). 서비스에 enableShare/disableShare/getByToken/importByToken 추가(토큰 randomBytes hex, import는 호출자 소유로 복사).
- 프론트: client share/unshare/getShared/import 메서드. TemplatePicker 행에 🔗(공유 링크 복사). 공개 페이지 `/template/:token`(TemplateSharePage) = 읽기전용 미리보기 + 로그인 시 "내 템플릿으로 가져오기".
- **버그 수정**: 비보안 HTTP(dev.brainsp.com)에서 `navigator.clipboard`가 undefined라 복사 시 예외 → 안내 alert 미표시. `util/clipboard.ts` 안전 복사 유틸(secure면 clipboard API, 아니면 execCommand 폴백, 예외 없음) 추가하고 템플릿/프로젝트 공유 양쪽에 적용.
- 검증: API 2계정 흐름(A 공유→공개 미리보기→B 가져오기→B 목록, 해제 후 404). 헤드리스(공유 버튼→토큰→공유 페이지 미리보기+가져오기 버튼). 에러 0.
- 프로덕션 재배포(웹 해시 index-CrZt2bv2.js).
